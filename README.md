## О проекте

**CookBookAI** — Android-приложение, объединяющее электронную кулинарную книгу и инструменты интеллектуального анализа продуктов питания.

Пользователь может создавать и хранить собственные рецепты, добавлять блюда в избранное, анализировать фотографию еды, получать предполагаемое название блюда и значения КБЖУ, а также сканировать штрихкоды готовых продуктов.

## Основные возможности

- регистрация и авторизация пользователей;
- создание, просмотр, редактирование и удаление рецептов;
- поиск и фильтрация рецептов;
- добавление рецептов в избранное;
- синхронизация пользовательских данных через Supabase;
- локальное хранение данных с использованием Room;
- работа в соответствии со стратегией Offline-first;
- загрузка фотографии блюда из галереи;
- отправка изображения на ML-сервер;
- распознавание категории или названия блюда;
- отображение Top-3 предсказаний и уровня уверенности модели;
- расчёт калорий, белков, жиров и углеводов;
- сохранение истории ИИ-анализов;
- сканирование штрихкодов через CameraX и Google ML Kit;
- добавление неизвестных продуктов вручную;
- пересчёт КБЖУ с учётом выбранного веса продукта;
- просмотр статистики питания;
- построение графиков;
- экспорт данных и рецептов в PDF;
- поддержка светлой и тёмной темы.

## Архитектура

Проект использует трёхкомпонентную архитектуру:

### Android-клиент

Отвечает за пользовательский интерфейс, локальное хранение, работу с камерой, отправку изображений на сервер и отображение результатов анализа.

### Supabase

Используется для:

-регистрации и авторизации;
-хранения профилей пользователей;
-облачного хранения рецептов и общих данных;
-синхронизации данных между устройствами;
-ограничения доступа к данным с помощью Row Level Security.

### FastAPI ML-сервер

Внешний серверный компонент выполняет обработку изображений:

-принимает фотографию от Android-клиента;
-обнаруживает объекты еды с помощью YOLOv8;
-классифицирует блюдо с помощью EfficientNetB0;
-формирует Top-3 предсказаний;
-рассчитывает приблизительные значения КБЖУ;
-возвращает результат в формате JSON.

## Локальные и облачные данные

### Room

Локально могут храниться:

-история ИИ-анализов;
-URI или путь к фотографии;
-результаты распознавания;
-значения КБЖУ;
-уровень уверенности;
-Top-3 предсказаний;
-пользовательские продукты со штрихкодами;
-кэш рецептов и данные для отложенной синхронизации.

### Supabase

В облаке хранятся:

-профили пользователей;
-рецепты;
-продукты и значения КБЖУ;
-пользовательские продукты;
-данные, необходимые для синхронизации.
-ER-диаграммы и варианты схем баз данных находятся в каталоге docs.
